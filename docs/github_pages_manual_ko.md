# GitHub Pages 설명서

GitHub Pages는 GitHub 저장소에서 직접 웹사이트를 호스팅할 수 있게 해주는 서비스입니다. 정적 웹사이트(HTML, CSS, JavaScript)를 무료로 호스팅하기에 매우 유용합니다.

## GitHub Pages란?

-   **정적 사이트 호스팅:** HTML, CSS, JavaScript 파일로 구성된 웹사이트를 위한 서비스입니다. Hugo, Jekyll, Gatsby 등 정적 사이트 생성기(SSG)로 만든 사이트를 호스팅하는 데 널리 사용됩니다.
-   **무료 호스팅:** 공개 저장소(public repository)에 대해서는 무료로 제공됩니다.
-   **저장소 연동:** `[username].github.io` 형태의 사용자 사이트나, 각 프로젝트 저장소별 사이트를 만들 수 있습니다.

## 작동 방식

GitHub Pages는 특정 브랜치의 콘텐츠를 웹사이트로 게시합니다. 이 과정을 자동화하기 위해 **GitHub Actions**를 사용하는 것이 일반적입니다.

### 소스 브랜치

웹사이트를 게시할 소스로 사용될 브랜치를 지정합니다. 과거에는 `gh-pages`라는 별도의 브랜치를 사용하는 경우가 많았지만, 최근에는 GitHub Actions를 통해 빌드된 결과를 바로 배포하는 방식이 더 선호됩니다.

### 빌드 및 배포

1.  **소스 코드 푸시:** 개발자는 소스 코드(예: Hugo 프로젝트)를 `main` 브랜치에 푸시합니다.
2.  **GitHub Actions 트리거:** 푸시 이벤트를 감지하여 미리 정의된 워크플로우(`.github/workflows` 안의 `yaml` 파일)가 실행됩니다.
3.  **빌드:** 워크플로우는 가상 환경에서 Hugo를 설치하고, `hugo` 명령을 실행하여 `public` 디렉토리에 정적 파일들을 생성합니다.
4.  **배포:** 생성된 `public` 디렉토리의 콘텐츠를 GitHub Pages에 배포합니다. 이 과정이 완료되면 지정된 URL에서 웹사이트를 볼 수 있게 됩니다.

## 이 저장소에서의 사용

이 `61315gamelab.github.io` 저장소는 **사용자 사이트**로 구성되어 있습니다.

-   **저장소 이름:** `[username].github.io` 형식의 이름을 가지므로, `main` 브랜치의 콘텐츠가 빌드되어 `https://61315gamelab.github.io` 주소에 바로 게시됩니다.
-   **워크플로우:** `.github/workflows/hugo.yaml` 파일에 정의된 워크플로우가 모든 배포 과정을 자동으로 처리합니다.
-   **소스 브랜치:** `main` 브랜치가 소스 코드의 역할을 합니다.

따라서 이 저장소의 경우, **`main` 브랜치에 변경 사항을 `push`하는 것만으로 웹사이트가 자동으로 업데이트됩니다.**

## 배포 확인하기

1.  GitHub 저장소 페이지에서 **'Actions'** 탭을 클릭합니다.
2.  가장 최근에 실행된 워크플로우(일반적으로 커밋 메시지와 함께 표시됨)를 클릭하여 진행 상황을 확인합니다.
3.  워크플로우의 'build'와 'deploy' 작업이 모두 녹색 체크 표시로 완료되면 배포가 성공한 것입니다.
4.  배포가 완료된 후 몇 분 정도 지나야 실제 웹사이트에 변경 사항이 반영될 수 있습니다. (캐시 문제)

## 주의사항

-   **정적 사이트만 가능:** PHP, Python, 데이터베이스 등 서버 측 코드는 실행할 수 없습니다.
-   **저장소 공개 여부:** 사용자/조직 사이트는 비공개(private) 저장소에서도 GitHub Pages를 무료로 사용할 수 있지만, 일반 프로젝트 사이트는 공개(public) 저장소여야 무료입니다.
