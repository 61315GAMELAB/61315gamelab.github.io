## 저장소 구조

이 저장소는 Hugo를 사용하여 정적 웹사이트를 생성하도록 구성되어 있습니다. 주요 디렉토리와 파일은 다음과 같습니다.

-   `archetypes`: `hugo new` 명령 사용 시 생성될 파일의 템플릿입니다.
-   `assets`: SCSS, JavaScript 등 처리나 번들이 필요한 파일들이 위치합니다.
-   `content`: 웹사이트의 모든 콘텐츠가 마크다운 파일 형태로 저장됩니다.
-   `config`: 웹사이트의 전반적인 설정을 담고 있습니다.
-   `static`: 이미지, 폰트 등 별도의 처리 없이 그대로 복사될 정적 파일들이 위치합니다.
-   `themes`: 웹사이트의 디자인과 레이아웃을 정의하는 테마가 포함됩니다. 현재 `hugo-theme-stack` 테마를 사용하고 있습니다.
-   `hugo.toml`: Hugo의 기본 설정 파일입니다.
-   `.github/workflows/hugo.yaml`: GitHub Actions를 이용한 자동 배포 워크플로우 정의 파일입니다.

## 콘텐츠 관리

### 콘텐츠 유형

모든 콘텐츠는 `content/post` 디렉토리 아래에 생성됩니다. 게시글은 목적에 따라 다음 세 가지 하위 디렉토리 중 하나에 위치합니다.

-   `activity`: 동아리 활동 기록
-   `meeting`: 정기 회의 및 기타 회의록
-   `project`: 동아리에서 진행하는 프로젝트 소개

### 새로운 글 추가하기

1.  **새로운 글 생성:**
    `hugo new` 명령을 사용하여 콘텐츠의 기본 파일을 생성합니다. 예를 들어, `2025년 12월 23일`의 활동 기록을 추가하려면 다음 명령을 실행합니다.

    ```bash
    hugo new post/activity/251223/index.md
    ```
    > **참고:** 날짜를 기반으로 `YYMMDD` 형식의 디렉토리를 만드는 것이 일반적인 규칙입니다. `index.md`라는 이름으로 파일을 생성하면 해당 디렉토리 자체가 하나의 페이지가 됩니다.

2.  **콘텐츠 작성:**
    생성된 `content/post/activity/251223/index.md` 파일을 텍스트 편집기로 엽니다. 파일 상단의 프론트 매터(Front Matter)를 수정하고, 그 아래에 마크다운 형식으로 내용을 작성합니다.

    ```yaml
    ---
    title: "여기에 제목을 입력하세요"
    date: 2025-12-23
    # 기타 필요한 메타데이터 (예: tags, categories)
    ---

    여기에 본문을 마크다운으로 작성합니다.
    ```

### 글에 이미지 추가하기

이미지는 해당 글의 `index.md` 파일과 **같은 디렉토리**에 위치시키는 것이 가장 좋습니다. 이렇게 하면 콘텐츠와 리소스를 함께 관리하기 용이합니다.

1.  이미지 파일을 `content/post/activity/251223/` 디렉토리에 복사합니다.
2.  마크다운 본문에서 다음과 같이 이미지를 삽입합니다.

    ```markdown
    ![대체 텍스트](image.jpg)
    ```

## 로컬에서 웹사이트 실행하기

1.  **저장소 복제 및 이동:**
    ```bash
    git clone https://github.com/61315gamelab/61315gamelab.github.io.git
    cd 61315gamelab.github.io
    ```

2.  **테마 서브모듈 초기화:**
    이 프로젝트는 테마를 Git 서브모듈로 관리합니다. 다음 명령으로 서브모듈을 가져옵니다.
    ```bash
    git submodule update --init --recursive
    ```

3.  **Hugo 서버 실행:**
    ```bash
    hugo server
    ```

4.  브라우저에서 `http://localhost:1313`으로 접속하여 사이트를 확인합니다.

## 변경 사항 게시하기 (배포)

1.  **변경 사항 추가:**
    ```bash
    git add .
    ```
2.  **변경 사항 커밋:**
    ```bash
    git commit -m "feat: 새로운 활동 기록 추가"
    ```
    > **커밋 메시지 규칙:** `타입: 제목` 형식(예: `feat:`, `fix:`, `docs:`)으로 작성하면 변경 내역을 파악하기 좋습니다.

3.  **원격 저장소로 푸시:**
    ```bash
    git push origin main
    ```

`main` 브랜치로 푸시하면, GitHub Actions가 자동으로 사이트를 빌드하고 GitHub Pages에 배포합니다. 배포 과정은 저장소의 'Actions' 탭에서 확인할 수 있습니다.
