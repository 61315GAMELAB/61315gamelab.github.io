# 61315 GAMELAB 웹사이트 관리 안내서

안녕하세요! 61315 GAMELAB 웹사이트 저장소에 오신 것을 환영합니다.

이곳은 동아리 웹사이트의 소스 코드를 관리하는 공간입니다. 이 안내서는 웹사이트를 처음 다루는 분들을 위해 필요한 정보와 각종 설명서의 위치를 안내하는 길잡이 역할을 합니다.

## 무엇부터 시작해야 할까요?

웹사이트를 수정하고 관리하기 위해서는 몇 가지 기본적인 도구와 개념을 알아야 합니다. 아래의 순서대로 문서를 읽어보시는 것을 추천합니다.

1.  **[Git 사용법](./docs/git_manual_ko.md)**
    -   Git은 소스 코드의 변경 이력을 관리하는 '버전 관리 시스템'입니다. 여러 사람이 함께 작업하거나 변경 사항을 추적하는 데 필수적인 도구입니다. 이 문서에서는 Git의 기본 개념과 가장 자주 사용하는 명령어를 안내합니다.

2.  **[마크다운 문법 설명서](./docs/markdown_manual_ko.md)**
    -   웹사이트의 모든 글은 마크다운(Markdown)이라는 간단한 형식으로 작성됩니다. 이 문서를 통해 글씨를 굵게 하거나, 목록을 만들고, 링크나 이미지를 넣는 등 다양한 서식을 적용하는 방법을 배울 수 있습니다.

3.  **[Hugo 사용법](./docs/hugo_manual_ko.md)**
    -   우리 웹사이트는 'Hugo'라는 도구를 사용해 만들어졌습니다. Hugo는 여러분이 마크다운으로 작성한 글을 실제 웹사이트 페이지(HTML)로 변환해주는 역할을 합니다. 이 문서에서는 Hugo의 기본적인 사용법을 설명합니다.

4.  **[GitHub Pages 설명서](./docs/github_pages_manual_ko.md)**
    -   GitHub Pages는 우리가 만든 웹사이트를 인터넷에 무료로 게시해주는 서비스입니다. 우리가 어떻게 웹사이트를 배포하고 있는지 그 원리를 이해할 수 있습니다.

5.  **[우리 저장소 설명서](./docs/repository_manual_ko.md)**
    -   모든 기본 지식을 익혔다면, 이제 우리 웹사이트에 맞게 콘텐츠를 추가하고 관리하는 구체적인 방법을 알아볼 차례입니다. 새로운 활동 기록이나 프로젝트 소개 글을 어떻게 작성하고 게시하는지 상세히 안내합니다.

## 요약

-   **Git**으로 코드를 관리하고,
-   **마크다운**으로 글을 작성하며,
-   **Hugo**로 사이트를 만들고,
-   **GitHub Pages**를 통해 인터넷에 게시합니다.

각 문서들을 차근차근 읽어보시면 웹사이트를 관리하는 전체적인 흐름을 파악하실 수 있을 것입니다. 궁금한 점이 있다면 언제든지 동아리 내 다른 부원들에게 질문해주세요!
